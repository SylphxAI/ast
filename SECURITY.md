# Security Policy

Report suspected vulnerabilities through GitHub Security Advisories for this
repository or by contacting `security@sylphx.ai`.

Do not open public issues for vulnerabilities, generated-parser supply-chain
weaknesses, maintainer tokens, publish credentials, private consumer source
code, or unreleased parser compatibility data. If a secret or package-publish
credential is exposed, rotate it first, then document the recovery evidence in
the owning runbook or incident record.
